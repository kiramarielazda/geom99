Open Street Map, Leaflet and KMZ files

Overview:

The use of Open Street Map (OSM) and leaflet have been a suitable combination for web trail mapping needs. Open street maps provides a variety of different base maps inclusive with an array of base data not always seen in other open-source base maps. Combined with the simplicity of Leaflet's open-source JavaScript Library web mapping has never been easier.

The Process:

What Works:

  • Displaying an OpenStreetMap base map from a URL.
  • Having an OpenStreetMap base map and using Leaflet’s JavaScript library to display the KMZ filesstored on the GitHub repository.
  • Displaying Pop ups for the KMZ point markers.
  • The ability to toggle layers on and off.
  • Overriding KMZ GPS Styling of point markers to web displayable markers.

What Doesn’t Work:

Task 1: Hosting a tiled server over GitHub to host OpenStreetMap Base tiles.

Result 1:The docker file would not set up correctly to host a server for the OSM tile(s).

Result 2: Orange “polygon” reflecting from the bends on the Dumoine Trail KMZ file.

Working with KMZ files:

KMZ files have been seeing their way out over the years and there is a reason why. After extensive testing with the original GPS KMZ file, in multiple platforms it proved to be extremely difficult and arose more problems than working with the GeoJSON, in similar platforms. The KMZ file needed many tweaks to get working with Leaflet and Open Street Map, such as adjusting the markers from the GPS markers, the trail having shadowing around the bends, while the GeoJSON was able to adjust the markers with ease and never had a shadowing issue around the trail bends. The KMZ files in QGIS proved to be just as difficult as the pop ups would not use any other field than ID or FID, whereas shapefiles and other formats were able to display all fields in the pop ups.

QGIS

Overview:

QGIS, being an open-source program with a rich variety of functionality, was worth looking into for base mapping needs. The program proved to be difficult to produce an effective functional Web map. As seen in the above section working with KMZ files is not ideal and many errors in QGIS were due to working with the KMZ files. After testing similar processes with a shapefile, many of the program crashes remained. With the only difference being the pop ups allowed more than the ID or FID field while working with shapefiles.

The Process:

What Works:

  • Desktop application: Importing data, geoprocessing, styling, and symbology.
  • QGIS2Web Plugin: Downloadable plugin to create Web maps using either OpenLayers, Leaflet, or Mapbox GL JS as outputs.
  • Pop ups: Creating pop up boxes for layers (Widgets in QGIS).
  • Exporting a web map with the OpenLayers extension.
  • Exporting a web map with the MapBox GL JS extension.
  • Exporting and Hosting a QGIS leaflet exported map with the KMZ markers rendered as a “single value” and shapefiles (waterbody, river, road, tote road trail).

What Doesn’t Work:

Task 1: Exporting the KML/KMZ styled Tote Road Trail markers as “unique values” by “name” as a Leaflet map.

Result 1: The Python script would fail each time, due to legend styling and crash QGIS desktop application. Tried: Removing all legends, and current styling to single values.

Result 1.1: The Python script would fail each time, due to a legend styling and crash QGIS desktop application. The result only changed if the KMZ files were completely removed from the project and brought back in and styled as single values.

Task 2: Importing the exported OpenLayers Web map or MapBox GL JS Web map and importing them into GitHub.

Result 2: GitHub would not accept the folders containing the styling for the OpenLayers Web map or the Markers for MapBox GL JS Web map as there were too many styling options (Over 100). Another issue being some files wouldn’t import to GitHub as they were “hidden”.

Task 3: Inserting specified text into pop ups on KMZ files.

Result 3: Only the ID or FID fields could be used from the KMZ files for the Web map popups, and the pop ups for the shapefiles weren't easy to filter through.
