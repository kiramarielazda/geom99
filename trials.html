QGIS ⭐⭐

Overview:

QGIS, being an open-source program with a rich variety of functionality, was worth looking into for base mapping needs. The program proved to be difficult to produce an effective functional Web map.

The Process:

What Works:

  • Desktop application: Importing data, geoprocessing, styling, and symbology.
  • QGIS2Web Plugin: Downloadable plugin to create Web maps using either OpenLayers, Leaflet, or Mapbox GL JS as outputs.
  • Pop ups: Creating pop up boxes for layers (Widgets in QGIS)
  • Exporting a web map with the OpenLayers extension
  • Exporting a web map with the MapBox GL JS extension
  • Exporting and Hosting a QGIS leaflet exported map with the KMZ markers rendered as a “single value” and shapefiles (waterbody, river, road, tote road trail).

What Doesn’t Work:

Task 1: Exporting the KML/KMZ styled Tote Road Trail markers as “unique values” by “name” as a Leaflet map.

Result 1: The Python script would fail each time, due to legend styling and crash QGIS desktop application. Tried: Removing all legends, and current styling to single values

Result 1.1: The Python script would fail each time, due to a legend styling and crash QGIS desktop application. The result only changed if the KMZ files were completely removed from the project and brought back in and styled as single values.

Task 2: Importing the exported OpenLayers Web map or MapBox GL JS Web map and importing them into GitHub.

Result 2: GitHub would not accept the folders containing the styling for the OpenLayers Web map or the Markers for MapBox GL JS Web map as there were too many styling options (Over 100). Another issue being some files wouldn’t import to GitHub as they were “hidden”.

Task 3: Inserting specified text into pop ups on KMZ files

Result 3: Only the ID or FID fields could be used from the KMZ files for the Web map popups, and the pop ups for the shapefiles weren't easy to filter through
